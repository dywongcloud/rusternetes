Architecture

Rusternetes follows the standard Kubernetes architecture with five core components, all written in safe Rust. They can run as separate containers in a cluster or as concurrent tokio tasks in a single process. 

        

          
          
          
          architecture
        

        
Control Plane
  api-server          Axum HTTPS REST API (76 handlers, 2500-line router)
  scheduler           Filter/score plugin scheduling
  controller-manager  31 reconciliation controllers

Node Components
  kubelet             Container runtime via bollard (Docker API)
  kube-proxy          iptables service routing

Storage
  etcd                Production — distributed key-value store
  SQLite (rhino)      Lightweight — embedded or gRPC server
  memory              Testing only

Key Features

  • Full Kubernetes REST API — core/v1, apps/v1, batch/v1, networking, storage, RBAC, and 15+ more API groups
  • 80+ resource types — Pods, Deployments, StatefulSets, Services, Secrets, CRDs, and more
  • Watch API — real-time event streaming with Server-Sent Events, bookmarks, and HTTP/2
  • Custom Resource Definitions — full CRUD, watch, status/scale subresources, schema validation
  • 12 volume types — emptyDir, hostPath, PVC, configMap, secret, projected, NFS, CSI, and more
  • RBAC — Roles, ClusterRoles, Bindings, ServiceAccount tokens (RS256 JWT)
  • Admission webhooks — validating, mutating, and ValidatingAdmissionPolicy (CEL)
  • Probes — liveness, readiness, startup with HTTP, TCP, exec, and gRPC handlers
  • Three deployment modes — full cluster, SQLite cluster, or single binary
  • 90% Kubernetes conformance — tested against the official K8s v1.35 e2e test suite
