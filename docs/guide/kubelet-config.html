Kubelet Configuration

The kubelet is the node agent that manages container lifecycle via the Docker API (bollard). It supports CLI flags, a YAML configuration file, and environment variables with a clear precedence order.

CLI Flags

Flag Type Default Description
--node-name String required Name of this node (must be unique in the cluster)
--etcd-servers String http://localhost:2379 Comma-separated list of etcd endpoints
--config String — Path to KubeletConfiguration YAML file
--root-dir String cwd or /var/lib/kubelet Root directory for managing kubelet files
--volume-dir String {root-dir}/volumes Directory path for managing volume data
--volume-plugin-dir String /usr/libexec/kubernetes/kubelet-plugins/volume/exec Directory where volume plugins are installed
--log-level String info Log level: trace, debug, info, warn, error
--sync-interval u64 10 Pod sync interval in seconds
--metrics-port u16 8082 Metrics and configz server port
--cluster-dns String auto-discovered Cluster DNS service IP. Auto-discovered from kube-dns Service if not set.
--cluster-domain String cluster.local Cluster domain suffix for DNS
--network String rusternetes-network Docker network to connect pods to
--storage-backend String etcd Storage backend: etcd or sqlite
--data-dir String ./data/rusternetes.db SQLite database path (only with --storage-backend=sqlite)

KubeletConfiguration File

The kubelet can load configuration from a YAML file specified with --config. The file format follows the Kubernetes kubelet.config.k8s.io/v1beta1 API: 

        

          
          
          
          kubelet-config.yaml
        

        apiVersion: kubelet.config.k8s.io/v1beta1
kind: KubeletConfiguration
rootDir: /var/lib/kubelet
volumeDir: /var/lib/kubelet/volumes
volumePluginDir: /usr/libexec/kubernetes/kubelet-plugins/volume/exec
syncFrequency: 10
metricsBindPort: 8082
logLevel: info
clusterServiceCidr: 10.96.0.0/12
Field Type Description
apiVersion String Must be kubelet.config.k8s.io/v1beta1
kind String Must be KubeletConfiguration
rootDir String Root directory for kubelet files
volumeDir String Directory for volume data
volumePluginDir String Directory for volume plugins (FlexVolume)
syncFrequency u64 Pod sync interval in seconds (must be > 0)
metricsBindPort u16 Port for the metrics server
logLevel String Log verbosity level
clusterServiceCidr String Service CIDR range. First IP becomes the kubernetes service host.

Environment Variables

Variable Maps To Description
KUBELET_ROOT_DIR rootDir Root directory for kubelet files
KUBELET_VOLUMES_PATH volumeDir Volume data directory
KUBELET_VOLUME_PLUGIN_DIR volumePluginDir Volume plugin directory
CLUSTER_SERVICE_CIDR clusterServiceCidr Service CIDR for deriving kubernetes service IP
KUBERNETES_SERVICE_HOST_OVERRIDE — Override the kubernetes service host IP. Used in Docker Desktop where the ClusterIP is not routable from bridge containers.
RUST_LOG logLevel Log level (standard Rust tracing env var)

Configuration Precedence

When the same setting is specified in multiple places, the kubelet resolves it with this precedence (highest wins): 

        

          
          
          
          precedence
        

        CLI flags           # highest priority
Config file         # --config kubelet-config.yaml
Environment vars    # KUBELET_ROOT_DIR, etc.
Built-in defaults   # lowest priority

RuntimeConfig Defaults

After merging all configuration sources, these defaults apply for any unset values:

Setting Default Value
Sync frequency 10 seconds
Metrics port 8082
Cluster DNS 10.96.0.10 (auto-discovered from kube-dns)
Cluster service CIDR 10.96.0.0/12
Volume plugin dir /usr/libexec/kubernetes/kubelet-plugins/volume/exec
Log level info

Service CIDR to Kubernetes Service IP

The kubelet derives the kubernetes service ClusterIP by taking the first usable IP address from the service CIDR range. This IP is injected into pods as the KUBERNETES_SERVICE_HOST environment variable. 

        

          
          
          
          CIDR derivation
        

        # Service CIDR       -> Kubernetes Service IP
10.96.0.0/12        -> 10.96.0.1
10.0.0.0/16         -> 10.0.0.1
192.168.0.0/24      -> 192.168.0.1
DOCKER DESKTOP OVERRIDE

In Docker Desktop environments, the ClusterIP (10.96.0.1) is not routable from bridge containers because kube-proxy's iptables DNAT only applies in the host network namespace. Set KUBERNETES_SERVICE_HOST_OVERRIDE to the API server's direct container IP instead. The TLS certificate includes Docker bridge IPs as SANs.

Kubelet API Endpoints

The kubelet exposes a local API server on the metrics port:

Endpoint Method Description
/metrics GET Prometheus metrics
/configz GET Current kubelet configuration (JSON)
/exec/:container_id POST Execute a command in a container
