Installation

Build Rusternetes from source, understand feature flags, and set up your development environment. For a quick Docker-based setup without building from source, see the Quick Start.

Build from Source

Install the Rust toolchain

Rusternetes requires Rust 2021 edition. Install via rustup: 

        

          
          
          
          terminal
        

        curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env

Verify your installation:

        

          
          
          
          terminal
        

        rustc --version   # 1.75+ required
cargo --version

Clone and build

        

          
          
          
          terminal
        

        git clone https://github.com/calfonso/rusternetes.git
cd rusternetes

# Debug build (faster compilation, slower runtime)
cargo build

# Release build (slower compilation, optimized runtime)
cargo build --release

The workspace contains 10 crates. A full debug build takes 3–5 minutes; release builds take 8–15 minutes depending on hardware. Incremental builds after code changes are much faster.

Build individual components

You can build a single crate to save time during development: 

        

          
          
          
          terminal
        

        # Build the all-in-one binary
cargo build -p rusternetes

# Build individual components
cargo build -p rusternetes-api-server
cargo build -p rusternetes-scheduler
cargo build -p rusternetes-controller-manager
cargo build -p rusternetes-kubelet
cargo build -p rusternetes-kube-proxy
cargo build -p rusternetes-kubectl

Run tests

        

          
          
          
          terminal
        

        # Run all workspace tests
cargo test

# Run tests for a single crate
cargo test -p rusternetes-kubelet

# Run a specific test by name
cargo test test_pod_scheduling

# With stdout output
cargo test test_pod_scheduling -- --nocapture

# Lint and format (run before committing)
make pre-commit

Feature Flags

Rusternetes uses Cargo feature flags to control optional functionality at compile time.

Feature Default Description
sqlite On Enables the embedded SQLite storage backend via rhino. Required for the all-in-one binary. 
        

          
          
          
          terminal
        

        # Build with default features (includes SQLite)
cargo build -p rusternetes

# Build without SQLite (etcd-only)
cargo build -p rusternetes --no-default-features

# Explicitly enable SQLite
cargo build -p rusternetes --features sqlite
SQLite and rhino

The sqlite feature pulls in rhino, a gRPC-compatible SQLite storage backend that implements the same Storage trait as etcd. It can run embedded in the same process (all-in-one mode) or as a standalone gRPC server. See Storage Backends for details.

Docker Images

Each component has its own Dockerfile for containerized deployment. The docker compose build command builds all of them.

Dockerfile Binary Description
Dockerfile.api-server api-server Axum HTTPS REST API
Dockerfile.scheduler scheduler Pod scheduler with filter/score plugins
Dockerfile.controller-manager controller-manager 31 reconciliation controllers
Dockerfile.kubelet kubelet Node agent with Docker runtime
Dockerfile.kube-proxy kube-proxy iptables service routing
Dockerfile.kubectl kubectl Built-in Rust kubectl
Dockerfile.rhino rhino Standalone SQLite storage server
Dockerfile rusternetes All-in-one binary

To build individual images:

        

          
          
          
          terminal
        

        # Build a single image
docker compose build api-server

# Build the all-in-one image directly
docker build -t rusternetes:latest -f Dockerfile .

All-in-One Binary

The rusternetes crate produces a single binary that runs the API server, scheduler, controller manager, kubelet, and kube-proxy as concurrent tokio tasks in one process. It defaults to embedded SQLite storage, so no external etcd is needed. 

        

          
          
          
          terminal
        

        # Build the all-in-one binary
cargo build -p rusternetes --release

# Run it (no Docker or etcd required)
./target/release/rusternetes

This is the simplest way to run a Rusternetes cluster — a single process, a single binary, no containers or external dependencies. See All-in-One Binary for configuration options and production usage.

When to use all-in-one vs Docker

The all-in-one binary is ideal for development, testing, and single-node deployments. For multi-node clusters or production workloads, use the Docker Compose setup with etcd for proper distributed state.

kubectl

Rusternetes includes a built-in kubectl implementation in the crates/kubectl crate. It implements the most common kubectl commands natively in Rust.

Built-in Rust kubectl

        

          
          
          
          terminal
        

        # Build the Rust kubectl
cargo build -p rusternetes-kubectl --release

# Run it
./target/release/kubectl get pods
./target/release/kubectl get nodes

System kubectl

The standard kubectl from the Kubernetes project works perfectly with Rusternetes. The API is fully compatible. Install it via your package manager or from the Kubernetes docs. 

        

          
          
          
          terminal
        

        # macOS (Homebrew)
brew install kubectl

# Linux (apt)
sudo apt-get install -y kubectl

# Linux (dnf)
sudo dnf install -y kubectl
Which kubectl to use?

Both the Rust and upstream Go kubectl work. The upstream Go version is recommended for day-to-day use since it has complete feature coverage. The built-in Rust kubectl is useful for development and for environments where you want a single Rust-only toolchain.

Platform Support

macOS

Runtime Status Notes
Docker Desktop Recommended Works out of the box on Intel and Apple Silicon
Podman Desktop Supported Requires rootful mode. CNI networking is not available in Podman Machine — the kubelet falls back to bridge networking automatically
macOS Sequoia and Podman

macOS Sequoia (15.x) introduced networking changes that can cause issues with Podman Machine. If you experience connection problems, try Docker Desktop or ensure Podman Machine is updated to the latest version.

Linux

Runtime Status Notes
Docker Engine Recommended Native Docker socket, full CNI support
Podman (rootful) Supported Must run as root or with rootful configuration. Rootless Podman is not supported due to network namespace restrictions.

Fedora / RHEL notes

Fedora ships Podman by default. To use Docker Engine instead: 

        

          
          
          
          terminal
        

        # Install Docker on Fedora
sudo dnf -y install dnf-plugins-core
sudo dnf config-manager --add-repo https://download.docker.com/linux/fedora/docker-ce.repo
sudo dnf install docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo systemctl start docker
sudo systemctl enable docker

# Add your user to the docker group
sudo usermod -aG docker $USER
newgrp docker

If you prefer Podman, ensure it runs in rootful mode for kube-proxy (which needs CAP_NET_ADMIN and host network access for iptables): 

        

          
          
          
          terminal
        

        # Run with rootful Podman
sudo podman compose up -d

Build dependencies

When building from source, you may need system libraries depending on your platform:

Platform Packages
Ubuntu / Debian build-essential pkg-config libssl-dev
Fedora / RHEL gcc openssl-devel pkg-config
macOS Xcode Command Line Tools (xcode-select --install) 
        

          
          
          
          terminal
        

        # Ubuntu / Debian
sudo apt-get install -y build-essential pkg-config libssl-dev

# Fedora / RHEL
sudo dnf install -y gcc openssl-devel pkg-config

# macOS (if not already installed)
xcode-select --install

Verify Your Installation

After building, confirm the binaries are present: 

        

          
          
          
          terminal
        

        # Check the all-in-one binary
./target/release/rusternetes --help

# Check individual components
./target/release/api-server --help
./target/release/scheduler --help
./target/release/controller-manager --help
./target/release/kubelet --help
./target/release/kube-proxy --help
./target/release/kubectl --help
