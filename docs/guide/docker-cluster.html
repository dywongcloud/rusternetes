Docker Compose with etcd

The full cluster deployment runs each Rusternetes component as a separate Docker container, backed by etcd for distributed state storage. This is the production-recommended deployment mode and the configuration used for conformance testing.

Services

The docker-compose.yml defines seven services that form the complete cluster:

Service Image Ports Network Description
etcd quay.io/coreos/etcd:v3.5 2379 (client), 2380 (peer) rusternetes-net Distributed key-value store for all cluster state
api-server rusternetes-api-server 6443 (HTTPS) rusternetes-net REST API with TLS, RBAC, webhooks, watch
scheduler rusternetes-scheduler — rusternetes-net Pod scheduling with affinity, taints, priority
controller-manager rusternetes-controller-manager — rusternetes-net 31 reconciliation controllers
node-1 rusternetes-kubelet 10250 (metrics) rusternetes-net Kubelet managing pod containers on node 1
node-2 rusternetes-kubelet 10251 (metrics) rusternetes-net Kubelet managing pod containers on node 2
kube-proxy rusternetes-kube-proxy — host iptables-based service routing (requires CAP_NET_ADMIN)
NOTE

The kube-proxy service runs in host network mode because it needs to manipulate the host's iptables rules. It requires the CAP_NET_ADMIN capability.

Ports

Port Protocol Service Description
6443 HTTPS api-server Kubernetes API endpoint (TLS). This is the port you point kubectl at.
2379 HTTP etcd etcd client API. Used internally by the API server.
2380 HTTP etcd etcd peer communication. Used for HA clustering.
10250 HTTP node-1 Kubelet metrics and health endpoint.
10251 HTTP node-2 Kubelet metrics and health endpoint (offset to avoid collision).

Environment Variables

Required

Variable Description Example
KUBELET_VOLUMES_PATH Absolute path on the host where pod volumes are stored. This directory is bind-mounted into the kubelet containers. Must be an absolute path. /home/user/rusternetes/.rusternetes/volumes

Optional

Variable Default Description
RUST_LOG info Log level for all Rust components. Accepts trace, debug, info, warn, error. Can be per-crate: rusternetes_api_server=debug,info.
DOCKER_HOST unix:///var/run/docker.sock Docker daemon socket path. Override if Docker uses a non-default socket location.
KUBERNETES_SERVICE_HOST_OVERRIDE — Override the API server address injected into pod environments. Used when pods need to reach the API server at a different address than the default service IP.

etcd Configuration

The etcd instance is configured for single-node operation with settings tuned for Kubernetes workloads:

Setting Value Description
auto-compaction-retention 10m Automatically compact the keyspace every 10 minutes. Prevents unbounded history growth from Kubernetes watch operations.
quota-backend-bytes 8589934592 (8 GB) Maximum database size. etcd raises an alarm and stops writes when this limit is reached.
snapshot-count 5000 Number of committed transactions between snapshots. Lower values trade disk I/O for faster recovery.
TIP

Monitor etcd database size with docker compose exec etcd etcdctl endpoint status --write-out=table. If the database grows beyond 4 GB, run a manual defragmentation: docker compose exec etcd etcdctl defrag.

Network

All services (except kube-proxy) communicate over the rusternetes-net bridge network. Docker assigns IPs from the bridge subnet (typically 172.18.0.0/16). The TLS certificates include SANs for these bridge IPs so that internal component-to-API-server communication works over HTTPS. 

        

          
          
          
          docker-compose.yml (network)
        

        
networks:
  rusternetes-net:
    driver: bridge
    name: rusternetes-net

Step-by-Step Setup

1. Set the volumes path

        

          
          
          
          terminal
        

        
# Must be an absolute path
export KUBELET_VOLUMES_PATH=$(pwd)/.rusternetes/volumes
mkdir -p $KUBELET_VOLUMES_PATH

2. Generate TLS certificates

        

          
          
          
          terminal
        

        
# Generates CA + server certs with Docker bridge IP SANs
bash scripts/generate-certs.sh
WARNING

The generated certificates include SANs for Docker bridge IPs (172.18.0.2-5). If your Docker network uses a different subnet, edit scripts/generate-certs.sh to match your environment before generating certs.

3. Build Docker images

        

          
          
          
          terminal
        

        
# First build takes ~10-15 minutes (Rust compilation)
# Subsequent builds are faster due to Docker layer caching
docker compose build

4. Start the cluster

        

          
          
          
          terminal
        

        
docker compose up -d

5. Bootstrap the cluster

        

          
          
          
          terminal
        

        
# Creates CoreDNS, kubernetes service, service account tokens
bash scripts/bootstrap-cluster.sh

6. Verify the cluster

        

          
          
          
          terminal
        

        
# Set kubeconfig
export KUBECONFIG=~/.kube/rusternetes-config

# Check nodes
kubectl get nodes
# NAME     STATUS   ROLES    AGE   VERSION
# node-1   Ready    <none>   30s   v1.35.0
# node-2   Ready    <none>   30s   v1.35.0

# Check system pods
kubectl get pods -n kube-system
# NAME                       READY   STATUS    RESTARTS   AGE
# coredns-xxxxxxxxxx-xxxxx   1/1     Running   0          30s

# Deploy a test workload
kubectl create deployment nginx --image=nginx:alpine
kubectl get pods -w

Operations

Viewing logs

        

          
          
          
          terminal
        

        
# All services
docker compose logs -f

# Single service
docker compose logs -f api-server

# With timestamps
docker compose logs -f --timestamps controller-manager

# Last 100 lines
docker compose logs --tail=100 scheduler

Rebuilding a single service

        

          
          
          
          terminal
        

        
# Rebuild and restart just the api-server
docker compose build api-server
docker compose up -d api-server

# Rebuild and restart kubelet node-1
docker compose build node-1
docker compose up -d node-1

Clean restart

        

          
          
          
          terminal
        

        
# Stop all services and remove containers
docker compose down

# Remove etcd data for a fresh start
docker volume rm rusternetes_etcd-data 2>/dev/null

# Remove pod volumes
rm -rf .rusternetes/volumes/*

# Rebuild and start fresh
docker compose build
docker compose up -d
bash scripts/bootstrap-cluster.sh

Checking etcd health

        

          
          
          
          terminal
        

        
# Endpoint status
docker compose exec etcd etcdctl endpoint status --write-out=table

# Endpoint health
docker compose exec etcd etcdctl endpoint health

# List all keys
docker compose exec etcd etcdctl get / --prefix --keys-only

# Count keys by resource type
docker compose exec etcd etcdctl get /registry/ --prefix --keys-only | \
  sed 's|/registry/\([^/]*\)/.*|\1|' | sort | uniq -c | sort -rn
DANGER

Never run docker compose down -v unless you intend to destroy all cluster state. The -v flag removes named volumes including etcd data.

Troubleshooting

Symptom Cause Fix
API server won't start etcd not ready yet Wait 5-10 seconds; api-server retries connection to etcd on startup
TLS certificate errors Docker bridge IPs changed Regenerate certs: bash scripts/generate-certs.sh
kube-proxy permission denied Missing CAP_NET_ADMIN Ensure the kube-proxy service has cap_add: [NET_ADMIN] in docker-compose.yml
Pods stuck in Pending Kubelets not registered Check kubectl get nodes; verify kubelets can reach the API server
Watch "context canceled" Missing HTTP/2 ALPN Verify TLS config includes h2 ALPN protocol; regenerate certs if needed
Volume mount failures KUBELET_VOLUMES_PATH not set Export the variable with an absolute path before running docker compose up
