Discovery Endpoints

The API server provides discovery endpoints that allow clients to enumerate available API groups, versions, and resources:

Endpoint Description
/api Lists core API versions
/api/v1 Lists all core v1 resource types
/apis Lists all API groups
/apis/{group}/{version} Lists resources in a specific API group version 
        

          
          
          
          terminal
        

        # List API groups
kubectl api-versions

# List all resource types
kubectl api-resources

# List resources in a specific group
kubectl api-resources --api-group=apps

Subresource Patterns

Many resource types support subresources that are accessed as nested paths beneath the parent resource:

Subresource Pattern Description
/status /{resource}/{name}/status Read/update the status subresource independently
/scale /{resource}/{name}/scale Read/update replica count (Deployments, ReplicaSets, StatefulSets)
/log /pods/{name}/log Stream container logs
/exec /pods/{name}/exec Execute a command in a container
/attach /pods/{name}/attach Attach to a running container
/portforward /pods/{name}/portforward Forward ports to a pod
/proxy /pods/{name}/proxy Proxy HTTP requests to a pod
/token /serviceaccounts/{name}/token Create a bound ServiceAccount token (RS256 JWT)
/binding /pods/{name}/binding Bind a pod to a node (used by the scheduler)
/eviction /pods/{name}/eviction Evict a pod (respects PodDisruptionBudgets)

        # Core API (v1) - namespaced

# Core API (v1) - cluster-scoped
/api/v1/{resource}
/api/v1/{resource}/{name}

# Named API groups - namespaced
/apis/{group}/{version}/namespaces/{namespace}/{resource}
/apis/{group}/{version}/namespaces/{namespace}/{resource}/{name}

# Named API groups - cluster-scoped
/apis/{group}/{version}/{resource}
/apis/{group}/{version}/{resource}/{name}

# List across all namespaces
/api/v1/{resource}
/apis/{group}/{version}/{resource}
