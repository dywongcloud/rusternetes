Deploy Your Way

Full Kubernetes in a multi-node cluster. Or in a single binary. Your call.

⚙

Multi-Node with etcd

The standard production deployment. Separate containers for each component, backed by an etcd cluster with Raft consensus. High availability with leader election, multi-master, and horizontal scaling.

docker compose up -d
🔀

Swap the Database

Don't want to run etcd? Drop in Rhino — an etcd-compatible gRPC server backed by SQLite, PostgreSQL, or MySQL. Same Kubernetes API, same binaries, zero etcd infrastructure. Just change the compose file.

docker compose -f docker-compose.sqlite.yml up -d
🚀

Single Binary, Single Process

All five Kubernetes components running as concurrent tokio tasks in one process with an embedded SQLite database. No containers. No external dependencies. Full Kubernetes API in a single binary.

./rusternetes --data-dir ./cluster.db

The all-in-one binary is purpose-built for environments where a full Kubernetes cluster is overkill: edge devices, CI/CD pipelines, local development, IoT gateways, embedded systems, and air-gapped environments. Your entire cluster state lives in a single SQLite file — back it up with cp, inspect it with sqlite3, move it to another machine.

Cargo Manifest

10 crates in the container yard. Modular Rust workspace, each crate ships independently.

api-server

Axum-based HTTPS REST API. Routes, handlers, auth, watch, webhooks.

controller-manager

31 reconciliation controllers: deployments, jobs, endpoints, GC, TTL, HPA, and more.

scheduler

Pod scheduling with filter/score plugins, affinity, taints, priority, preemption.

kubelet

Node agent. Container lifecycle via bollard (Docker API), volumes, probes, networking.

kube-proxy

Service routing via iptables. ClusterIP, NodePort, LoadBalancer. Host network mode.

storage

Pluggable storage: etcd, SQLite (via rhino), and in-memory backends. Resource versioning via mod_revision.

common

Shared resource types (Pod, Service, Deployment, etc.), error handling, utilities.

kubectl

CLI tool for cluster interaction. get, create, apply, delete, logs, exec, and more.

cloud-providers

AWS, GCP, and Azure integrations for LoadBalancer and cloud-native features.

rusternetes

All-in-one binary. All components as tokio tasks in a single process with embedded SQLite.

Quick Start

Get a local cluster running in minutes with Docker Compose.

terminal 
# Clone and build
git clone https://github.com/calfonso/rusternetes.git
cd rusternetes
cargo build --release

# Start the cluster with Docker Compose
export KUBELET_VOLUMES_PATH=$(pwd)/.rusternetes/volumes
docker compose build
docker compose up -d

# Bootstrap CoreDNS, services, and SA tokens
bash scripts/bootstrap-cluster.sh

# Use the cluster
export KUBECONFIG=~/.kube/rusternetes-config
kubectl get nodes
kubectl get pods -A

Why Rust?

Memory Safety

No garbage collector, no null pointer dereferences, no data races. Rust's ownership model eliminates entire classes of bugs that plague C/C++ and even Go control planes.

Performance

Zero-cost abstractions and no runtime overhead. Async I/O via Tokio delivers high throughput with minimal resource consumption compared to goroutine-based implementations.

Correctness

The type system catches mismatches at compile time. Serde ensures wire-compatible JSON serialization. 149 rounds of conformance testing validate behavior against real Kubernetes.
